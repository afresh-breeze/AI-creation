# 基础版跑酷（base）

这是一个轻量 3D 跑酷小游戏（基础版）：
你控制小球在三维赛道上左右移动，躲障碍、吃积分、冲分过关。

*虽然略显丑陋，但是还可能会有未知的bug！*

---

## 1) 游戏介绍

`base` 版本是项目里的基础玩法实现，主要是：

- 持续前进赛道；
- 随机关卡障碍与积分；
- 分数达标后自动过关；
- 前九关挑战 + 第十关无限模式。

*当你无论如何也打不到第十关时，或许就已经不在意它并不美丽的外表和并不丰富的功能了？*

---

## 2) 游戏玩法

- **PC 端移动**：`A / ←` 左移，`D / →` 右移。
- **移动端移动**：在屏幕下方左右滑动。
- **暂停**：`空格`（移动端可点击触摸区上方暂停）。
- **目标**：躲避红色障碍物，收集黄色发光积分，达到关卡目标分数。

*如果你用心就会发现临死前按下暂停并选择重新开始是损失最小化的做法！不过或许一头撞死也不错？*

## 3) 涉及技术

- **Three.js**：3D 场景、相机、几何体、材质、渲染与动画。
- **HTML**：UI 容器和游戏界面结构。
- **CSS**：样式、面板布局、响应式适配。
- **JavaScript**：输入控制、主循环、对象生成、碰撞判定、关卡与状态管理。

---

## 4) 核心功能实现

- **场景初始化**
  - 初始化 `scene / camera / renderer`；
  - 创建道路、小球、障碍物与积分系统；
  - 恢复本地存档状态并更新 UI。

- **动态生成系统**
  - 按路段实时刷障碍和积分；
  - 随关卡提升障碍概率和节奏密度；
  - 保持难度随进度递增。

- **碰撞与得分**
  - 障碍物命中：游戏结束；
  - 积分命中：加分并移除积分对象。

- **关卡切换与存档**
  - 分数达标后弹出过关窗口；
  - 进入下一关后清零本关分数并提升难度；
  - 支持通过 `localStorage` 保存基础进度。

- **暂停与重开逻辑**
  - 支持键盘/移动端暂停；
  - 区分“游戏结束重开”和“暂停重开”的处理路径。

## 6) 整体逻辑

运行流程可概括为：

1. 初始化场景与游戏状态；
2. 玩家控制小球左右移动；
3. 赛道持续滚动并动态生成障碍与积分；
4. 每帧判定碰撞并更新分数；
5. 达标则过关，不达标继续冲分；
6. 失败后可重开，继续挑战更高关卡。
